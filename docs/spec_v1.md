# じゃんけん・あっち向いてほいアプリ 要件定義書

## 目的

ユーザーが CLI または HTTP API から、以下の 2 種類のゲームを遊べるアプリを提供する。

1. じゃんけん単体
2. じゃんけんの勝敗後に「あっち向いてほい」を行う 2 段階勝負

どちらのゲームもセッション型で進行する。永続化は行わず、セッション状態はアプリケーション実行中のメモリ上で保持する。プロセス終了や再起動によってセッションが失われることは許容する。

## 提供するゲーム

### 1. じゃんけん

ユーザーとシステムがじゃんけんを行う。

ユーザーは `グー`、`チョキ`、`パー` のいずれかを入力する。システムはランダムに手を出す。

勝敗は次のいずれかになる。

- `ユーザーの勝ち`
- `ユーザーの負け`
- `あいこ`

ユーザーが勝った場合、または負けた場合はゲームを終了する。
あいこの場合は、同じセッションのまま再度じゃんけんを行う。

### 2. あっち向いてほい

あっち向いてほいは、じゃんけんと方向勝負の 2 段階で構成される。

まず、通常のじゃんけんを行う。
じゃんけんがあいこの場合は、同じセッションのまま再度じゃんけんを行う。

じゃんけんでユーザーが勝った場合、ユーザーが主導権を持つ。
このときユーザーは `上`、`下`、`右`、`左` のいずれかを指定する。システムが向いた方向と一致した場合、ユーザーの勝利とする。一致しない場合は、勝敗未決として新しいじゃんけんからやり直す。

じゃんけんでユーザーが負けた場合、システムが主導権を持つ。
このときユーザーは `上`、`下`、`右`、`左` のいずれかへ回避する。システムが指定した方向と一致した場合、ユーザーの敗北とする。一致しない場合は、勝敗未決として新しいじゃんけんからやり直す。

## セッション状態

### じゃんけん単体

じゃんけん単体のセッションは、次の状態を持つ。

- `waiting_for_user_hand`: ユーザーの手入力待ち
- `finished`: 勝敗確定済み

### あっち向いてほい

あっち向いてほいのセッションは、次の状態を持つ。

- `waiting_for_janken_hand`: じゃんけんの手入力待ち
- `waiting_for_look_direction`: 方向入力待ち
- `finished`: 最終勝敗確定済み

## 入力仕様

じゃんけんの手として受け付ける値は次の 3 つのみとする。

- `グー`
- `チョキ`
- `パー`

あっち向いてほいの方向として受け付ける値は次の 4 つのみとする。

- `上`
- `下`
- `右`
- `左`

ローマ字、英語、ひらがな、別名表記などの変換は行わない。
合意していない入力補完や代替解釈は行わない。

あっち向いてほいでは、現在の状態に応じて必要な入力が異なる。

- `waiting_for_janken_hand` のときは、じゃんけんの手のみを受け付ける
- `waiting_for_look_direction` のときは、方向のみを受け付ける
- 状態と入力が一致しない場合はエラーとする

## CLI 要件

CLI では、次の 2 つのコマンドを提供する。

- `janken`
- `acchi-muite-hoi`

`janken` は、じゃんけん単体の対話型アプリとして動作する。
起動後にセッションを開始し、ユーザーに `グー`、`チョキ`、`パー` の入力を促す。勝敗が決まったら、ユーザーの手、システムの手、結果を表示して終了する。あいこの場合は再入力を促す。

`acchi-muite-hoi` は、あっち向いてほいの対話型アプリとして動作する。
起動後にセッションを開始し、状態に応じてじゃんけんの手、または方向の入力を促す。最終勝敗が決まったら結果を表示して終了する。

CLI で不正な入力があった場合は、エラーメッセージを表示し、必要に応じて再入力を促す。

## HTTP API 要件

HTTP API では、Swagger UI から操作できることを前提とする。

じゃんけん単体では、次の API を提供する。

- `POST /sessions`
- `POST /sessions/{session_id}/hands`

`POST /sessions` は、じゃんけんセッションを開始する。
レスポンスには `session_id`、`status`、`message` を含める。

`POST /sessions/{session_id}/hands` は、ユーザーの手を受け取り、システムの手、勝敗、状態、メッセージを返す。あいこの場合は同じセッションを継続する。

あっち向いてほいでは、次の API を提供する。

- `POST /acchi-muite-hoi/sessions`
- `POST /acchi-muite-hoi/sessions/{session_id}/turns`

`POST /acchi-muite-hoi/sessions` は、あっち向いてほい用セッションを開始する。
レスポンスには `session_id`、`status`、`message` を含める。

`POST /acchi-muite-hoi/sessions/{session_id}/turns` は、現在の状態に応じて、じゃんけんの手または方向入力を受け取り、ゲームを 1 ステップ進める。

## エラー要件

次の場合はエラーとして扱う。

- 不正な手が入力された
- 不正な方向が入力された
- 状態と入力内容が一致しない
- 存在しない `session_id` が指定された
- すでに終了したセッションに対して入力された
- 想定外の例外が発生した

HTTP では、代表的なエラーを次のように扱う。

- 入力値の形式不正: `422`
- 存在しないセッション: `404`
- 終了済みセッションへの入力: `409`
- 状態と入力内容の不整合: `409`

想定外の例外は握りつぶさない。
合意していないフォールバック動作や、暗黙の代替処理は行わない。

## テスト要件

少なくとも次の観点を確認する。

- じゃんけんセッションを開始できること
- じゃんけんでユーザー勝ちを判定できること
- じゃんけんでユーザー負けを判定できること
- じゃんけんであいこの場合に同じセッションで継続できること
- あっち向いてほいのセッションを開始できること
- じゃんけん勝利後に方向入力へ進めること
- じゃんけん敗北後に方向入力へ進めること
- 方向が一致した場合に最終勝敗が確定すること
- 方向が一致しない場合に新しいじゃんけんへ戻ること
- 不正な手がエラーになること
- 不正な方向がエラーになること
- 状態と入力内容の不整合がエラーになること
- 存在しないセッションがエラーになること
- 終了済みセッションへの入力がエラーになること
- CLI で正常系と入力エラーを確認できること
- HTTP API で正常系とエラー変換を確認できること

ランダム性が関わる部分は、テスト時に結果を固定できるようにする。

## スコープ外

次の内容は今回のスコープに含めない。

- セッションの永続化
- 勝敗履歴の保存
- 勝敗履歴の集計
- ユーザー認証・認可
- ローマ字、英語、ひらがななどの別名入力対応
- 1 手ずつ実行する非対話型 CLI
- 複数ユーザー対戦
- 管理画面
